Keyboard interaction

  • Roving tabindex is used to manage focus. Only one calendar day is possible to tab to at any one time.

  • The first press on TAB reveals a link to keyboard shortcuts.

  • Keyboard shortcuts are displayed in a modal with a focus loop. Press OK or ESC to close modal.

  • Second press on TAB to enter the calendar. Focus is placed on the day that is selected. If no date is selected, first day in the month gets focus.

  • LEFT ARROW and RIGHT ARROW to move to previous day or next day.

  • UP ARROW and DOWN ARROW to move one week back or one week ahead

  • PAGE UP and PAGE DOWN to move one month back or one month ahead.

  • HOME to go to the first day in the month, and END to go to the last day of the month

  • Select a date using SPACE or RETURN keys

  • Leave the calendar with TAB or SHIFT + TAB

Screen reader features

  • Please note
    role='application' is used. This can cause serious problems for screen reader users if not implemented correctly. Proceed with caution!

  • Days are coded as <td role="button">

  • aria-label to set human-readable names on calendar days

    <td role='button' aria-label='Today Wednesday 20 February 2019'>

  • Selected day is indicated in its label:

    <td role='button' aria-label='Selected Today Wednesday 20 February 2019'>

  • Native calendar semantics is suppressed with role="presentation"

  • Table header with days shorthand is hidden from screen readers with aria-hidden

    <thead aria-hidden="true"><tr><th scope="col">Mo</th><th scope="col">Tu</th><th scope="col">We</th><th scope="col">Th</th><th scope="col">Fr</th></tr></thead>
